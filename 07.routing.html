Routing

There's three main options for routing in React:

  1. React Router
  2. Reach Router
  3. Navi

Reach Router

By default Reach Router adds some accessibility features, mainly around handling and triggering focus events, so we're gonna go with it.

  • Install: $ npm i @reach/router

Use:

  • <Link to="/"> generates an <a> element with the appropiate navigation logic to go to the specified path without triggering a page refresh
  • <Router path="/" > indicates the content area that will display the appropiate content depending on the route the user is currently in
    • You include each component to be displayed as a child element with a path attribute that should match the desired route, e.g. <HomeContent path="/" />
    • The path attribute can hold variables if you mark them with a colon character :, e.g. for the path details/:id, if the user goes to the route details/123, then the component will be displayed and it will receive a props key of id with a value of 123
    • When there's multiple components with paths that match the current route, Reach will only display the most specific one, e.g. if the current route matches both <DetailsPage1 path="/details/1" /> and <DetailsPageGeneric path="/details/:id" /> then DetailsPage1 will be displayed
